一个面向高校课程场景的 **作业管理 + AI 批改 + 教学助手** 平台，覆盖 **“布置作业 → 学生提交 → AI 先批 → 教师复核 → 成绩发布/分析”** 的完整闭环。  
让教师把时间用在教学本身，而不是机械批改与反复汇总；让学生更快获得**有依据、可追溯**的反馈。

## 为什么选择它——面向真实教学的“可落地”

- **AI 先批、教师可控**：AI 输出“建议评分 + 依据”，教师一键采纳或调整，最终成绩由教师确认。
- **全流程留痕可追溯**：作业发布后会固化题目/答案/细则快照，批改与发布过程可复盘、便于申诉处理。
- **课堂效率提升**：自动汇总提交进度、缺交名单、AI 批改状态，让“催交/统计/核对”从繁琐变清晰。
- **题库体系化**：支持教材-章节-题目结构，题库可按“学校授权范围”治理，便于规模化教学复用。
- **可商用落地**：支持院系/学校私有化部署与品牌定制（学校名称/登录页等），适合试点与推广。

## 适用对象与典型场景

**面向对象**
- 高校教师 / 助教：大班课程、过程性考核、统一评分标准课程
- 学生：需要快速反馈与历史成绩跟踪
- 教学管理者：班级导入、课程管理、题库治理与跨校授权

**典型场景**
- 计算/证明/简答类作业：需要“按细则给分 + 可解释扣分点”
- 大班教学：大量提交与成绩统计压力大
- 实训课/课程设计：希望更快看到共性问题与班级水平分布

## 功能概览（按角色）

### 学生端
- 查看课程与已发布作业
- 支持 **文字 + 图片** 提交答案（适配手写作答场景）
- 查看批改结果、历史成绩与个人表现趋势（以教师发布为准）
- 使用 AI 助手进行学习问答（课程/作业范围内）

### 教师端
- 课程与学生管理：添加/导入学生、维护班级名单
- 作业发布：题目、总分、各题权重、截止时间与学生端可见性策略
- 提交进度：已交/未交名单、批改状态一目了然
- 批改复核：查看 AI 建议分与依据，人工确认最终成绩并发布
- 课程成绩矩阵：按学生/作业汇总成绩，便于期末成绩核算
- 题库维护：导入与管理教材/章节/题目结构，沉淀可复用资源
- 教学助手：快速查询课程/作业/成绩等数据并生成总结（可选开启）

### 管理员端
- 班级导入与课程管理
- 题库治理：教材与题库可见范围支持“按学校授权”
- 管理教学资源与基础数据（便于院系规模化运维）

## 平台亮点

### 1) 价格低廉，功能丰富
- 需昂贵的商业软件采购费用，即可获得完整的作业管理与 AI 批改能力。
- 系统集成作业发布、AI 自动评分、教师复核、成绩统计、题库管理与教学助手等核心功能，帮助院系以更低成本实现教学数字化升级。
  
### 2) 评分标准可复制：从“人治”到“有章可循”
- 支持将评分细则固化到作业快照，减少不同助教/不同批次的评分漂移
- AI 输出结构化建议，降低复核时间成本

### 3) 过程管理可视化：交没交、批没批、发没发，一眼看懂
- 缺交名单、提交进度、批改状态统一呈现
- 成绩发布与学生可见策略可控（是否可看答案/分数等）

### 4) 题库资产化：从“临时出题”到“内容资产沉淀”
- 教材-章节-题目树结构，支持多题型
- 题库按学校授权范围治理，适合校内/跨校协作与内容运营

### 5) AI 助手：把“查数据、做汇总”变成对话
- 教师：快速了解班级成绩概况、作业完成情况、学生分布
- 学生：针对作业与课程内容进行学习问答与复盘

## 试用与部署

你可以把它当作一个“学校内部系统”来部署：  
- **默认支持私有化部署**（数据在校内/你自己的服务器）
- 可按院系/学校做 UI 文案与登录页定制（更像“正式系统”而非 Demo）

最简启动方式（开发/试用）：
1) 复制环境变量模板并填写关键项（数据库、模型 API Key 等）
2) 一条命令启动所有服务

```bash
git clone https://github.com/pythc/ai-homework-system.git
cd ai-homework-system
cp .env.example .env
docker compose up -d --build
````
